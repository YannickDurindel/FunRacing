# FunRacing

A local PC racing game built in **Godot 4.6.1**, inspired by the Formula 1 career mode of Real Racing 3. Your phone connects over local WiFi as a steering wheel — gyroscope for steering, touch buttons for throttle, brake, DRS, and ERS.

> **Status:** Career hub UI complete · Physics & AI systems scaffolded · Track meshes generated · Race gameplay in progress

---

## Features

### Career Hub
- **Season selector** — choose any F1 season from 2019 to 2026
- **Accurate calendars** — correct round counts, sprint weekends identified per year, proper session formats (2021 / 2022 / 2023+ sprint formats all differ)
- **Circuit-specific practice missions** — missions adapt to the circuit type:
  - 🏙 **Street** (Monaco, Baku, Singapore, Jeddah, Miami, Las Vegas) — barrier familiarisation, zero-margin hot laps
  - ⚡ **High-speed** (Monza, Spa, Silverstone, Suzuka, Zandvoort) — flat-out sectors, slipstream study, draft defence
  - 🔧 **Technical** (Hungaroring, Barcelona, Abu Dhabi, Bahrain, Qatar) — brake bias tests, sector 2 attacks
  - 🔄 **Mixed** (COTA, Australia, Canada, Brazil, etc.) — installation laps, tyre comparisons, dirty air tests

### Driver Selection
Pick from the real **2025/2026 F1 grid** — 20 drivers across 10 teams, with team colours, car numbers, and country flags. Your choice persists across sessions; no re-asking on relaunch.

### Car Physics
- `VehicleBody3D` at 740 kg, 120 Hz physics tick
- Downforce: `speed² × 3.0 N`, drag coefficient adjustable per upgrade
- **DRS** — reduces drag by 15% when active in detection zones
- **ERS** — 4000 kJ store, regenerates under braking, boost via central force
- **Tyre model** — compound grip curves, thermal degradation, cliff behaviour
- **Pit stop** state machine: APPROACHING → STOPPED (2.5 s) → COMPLETE

### AI System
- 19 fictional AI drivers across 10 teams, each with skill (0.60–0.95) and aggression ratings
- Spline-based racing line with 20 m lookahead steering
- Overtake logic triggered when gap < 2 m
- Predetermined pit windows with compound strategy
- Mild rubber-banding: AI target speed scaled by player championship position

### Phone Controller
The game runs on your PC. Your phone is the steering wheel.

1. Start the HTTPS server: `cd game/web_controller && python3 serve.py`
2. Open the displayed URL on your phone's browser
3. Tilt to steer (DeviceOrientation API), touch pedals for throttle/brake, tap buttons for DRS/ERS

WebSocket on port 8080. Requires HTTPS on LAN for gyro access — a self-signed cert is included.

### Audio
Fully procedural engine audio via `AudioStreamGenerator` — no external audio files. V10 character with fundamental + 2nd/4th/6th harmonics, pitch mapped to RPM.

## Getting Started

### Requirements
- [Godot 4.6.1](https://godotengine.org/download) (Forward+ renderer)
- Python 3 (for the phone controller HTTPS server)
- Blender 5.0+ headless (optional, to regenerate track/car meshes)

### Run the game
```bash
# Open the project in Godot
./Godot_v4.6.1-stable_linux.x86_64 --path game/

# Or headless test
./Godot_v4.6.1-stable_linux.x86_64 --headless --path game/ --quit
```

### Generate assets (optional)
```bash
cd game/tools
bash generate_assets.sh
# Outputs .glb files into cars/ and tracks/
```

### Phone controller
```bash
cd game/web_controller
python3 serve.py
# Displays the local HTTPS URL — open it on your phone
```

## Architecture Notes

| Singleton | Responsibility |
|-----------|----------------|
| `GameState` | Scene transitions, active profile/career reference |
| `RaceState` | Live race data for all 20 cars (positions, gaps, sectors) |
| `CareerManager` | Championship points, season advance, constructor standings |
| `SaveManager` | JSON load/save to `user://saves/career.json` |
| `ControllerServer` | WebSocket server, gyro normalisation, input signals |
| `AudioManager` | Procedural engine audio, RPM → pitch mapping |

**Key Godot 4 gotchas respected in this codebase:**
- `WebSocketMultiplayerPeer.create_server()` — not the deprecated `WebSocketServer`
- `VehicleWheel3D` must be direct children of `VehicleBody3D` (no nesting)
- `get_skidinfo()` is inverted: `0` = skidding, `1` = no skid
- `AudioStreamGeneratorPlayback.push_buffer()` — not `push_frames` (changed in 4.6)
- `JSON.parse_string()` — not the old `JSON.new().parse()` pattern
- All UI built in GDScript `_ready()` — no `@onready` node-path dependencies in `.tscn`

## Save Format

Profiles save to `user://profile.json` (driver identity, never re-asked).
Career progress saves to `user://saves/career.json` (schema v1):

```json
{
  "schema_version": 1,
  "player": { "name": "Max Verstappen", "team_id": "red_bull", "car_number": 1 },
  "season_number": 1,
  "upgrades": {
    "engine": { "level": 0 }, "aero": { "level": 0 },
    "brakes": { "level": 0 }, "tires": { "level": 0 }, "ers": { "level": 0 }
  },
  "prize_money": 0,
  "personal_bests": {}
}
```
